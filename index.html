使用Hugo+Github pages搭建个人网站

2019-11-03
计算机技术

使用Hugo+Github pages搭建个人网站

1. 安装Hugo

官网直接下载安装即可
- ubuntu示例：
* 选择指定版本的deb包，复制下载链接
* wget https://github.com/gohugoio/hugo/releases/download/v0.59.0/hugo_0.59.0_Linux-64bit.deb
* dpkg -i hugo*.deb
* 将hugo添加到PATH路径
* hugo version测试

2. 构建Hugo项目

1

hugo new site [project-name]

例如：

1

hugo new site jason-site

构建完成后，在jason-site文件夹下会生成以下目录结构：

1
2
3
4
5
6
7
8

.
├── archetypes # 新生成文章的模版
├── config.toml # 存放 hugo 配置文件
├── content # 存放 markdown 文件
├── data # 存放 Hugo 处理的数据
├── layouts # 存放布局文件
├── static # 存放静态文件 图片 CSS JS文件
└── themes # 存放主题

3. 添加主题

为快速搭建网站，可从Huto主题市场下载相关主题。以even主题为例：

1
2

cd jason-site
git clone https://github.com/olOwOlo/hugo-theme-even themes/even

4. 启动Hugo

将themes/even/exampleSite文件夹下的所有内容拷贝到jason-site根目录下，启动Hugo:

1

hugo server

浏览器打开http://localhost:1313浏览效果

5.配置主题

    为了满足个性需求，需要修改config.tom文件。

    进入themes/even文件夹，会发现目录结构和新建的Hugo项目目录结构几乎一致。这样设置是为了用户的配置可以覆盖掉主题的配置。
    比如，在项目目录下新建layouts/partials/footer.html文件，填入自定义内容，可以覆盖掉hugo-theme-even主题由themes/even/layouts/partials/footer.html底部配置定义的底部显示。

6.生成新的文章

1

hugo new posts/first-post.md

在content目录中会自动以archetypes/default.md为模板在content/posts目录下生成一篇名为first-post.md的文章草稿：

title: “First Post”
date: 2017-12-27T23:15:53-05:00
draft: true
